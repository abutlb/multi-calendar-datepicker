Built-in Dark Themes

theme: 'dark'

Custom Dark via CSS Variables (Midnight variant)

💡 You can scope dark theme vars to a specific wrapper class in CSS — this avoids affecting other pickers on the page. The .midnight-wrap .mcd-picker rule in this page's <style> block demonstrates this.
يمكن تطبيق الثيم على تقويم واحد فقط بتعريف المتغيرات على كلاس محدد.

Code

// Option A: Use a built-in named theme
MultiCalendarDatepicker.init('#d_dark', {
  calendar: 'gregorian',
  locale:   'en',
  theme:    'dark',      // 'dark' | 'midnight' | 'ocean' | 'warm' | ...
  outputTarget: '#d_dark_val',
});

// Option B: theme: 'auto' — matches the OS dark/light preference
MultiCalendarDatepicker.init('#my_picker', {
  theme: 'auto', // uses @media (prefers-color-scheme: dark)
});

// Option C: Inline theme object — full custom dark palette
MultiCalendarDatepicker.init('#my_picker', {
  theme: {
    vars: {
      '--mcd-bg':          '#0f172a',
      '--mcd-text':        '#e2e8f0',
      '--mcd-primary':     '#6366f1',
      '--mcd-primary-text':'#ffffff',
      '--mcd-day-text':    '#e2e8f0',  // must be explicit in dark themes!
      '--mcd-weekday-text':'#94a3b8',
      // ... more vars
    },
  },
});
⚠️ Dark theme rule: Always set --mcd-day-text, --mcd-today-text, --mcd-weekday-text, and --mcd-nav-text explicitly to a light color. Relying on var(--mcd-text) inheritance can fail in some browsers when vars are applied inline via ThemeManager.
قاعدة الثيم الداكن: حدد ألوان النصوص صراحةً في الثيمات الداكنة — لا تعتمد على الوراثة.
