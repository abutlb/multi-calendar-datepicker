اسم المشروع:
Multi Calendar Datepicker CDN Library

الهدف:
بناء مكتبة JavaScript احترافية وخفيفة يمكن إضافتها لأي موقع عبر CDN، وتحوّل أي input إلى Datepicker يدعم عدة تقاويم، مع حفظ القيمة النهائية دائمًا كتاريخ ميلادي Gregorian بصيغة مناسبة لقواعد البيانات.

بالإضافة إلى ذلك، يجب أن تدعم المكتبة إخراج النتيجة المحولة في عناصر ظاهرة مثل div أو span أو textarea، بحيث يمكن استخدامها أيضًا كأداة تحويل تقاويم مباشرة، وليس فقط كحقل داخل نموذج.

الفكرة الأساسية:
المستخدم يختار التاريخ من واجهة حسب نوع التقويم المطلوب، مثل الميلادي أو الهجري أو الفارسي، بينما المكتبة تقوم داخليًا بتحويل التاريخ المختار إلى تاريخ ميلادي ISO مثل YYYY-MM-DD وتضعه في حقل hidden أو تعرضه في عنصر output مخصص أو تعيده عبر JavaScript API.

المتطلبات الأساسية:
1. بناء مكتبة JavaScript Vanilla بدون الاعتماد على jQuery.
2. يمكن استخدامها عبر CDN بسهولة.
3. تشمل ملف JavaScript وملف CSS.
4. تدعم أكثر من تقويم.
5. تحفظ الناتج النهائي كتاريخ ميلادي.
6. تدعم إخراج النتيجة في عنصر ظاهر مثل div أو span أو textarea.
7. تدعم اللغة العربية والإنجليزية.
8. تدعم RTL و LTR.
9. تكون قابلة للتوسع لإضافة تقاويم جديدة لاحقًا.
10. لا تعتمد على backend.
11. تعمل في المتصفحات الحديثة.
12. يجب إنشاء صفحة تجربة Demo Page يستطيع المستخدم من خلالها اختبار المكتبة بنفسه.

أنماط الاستخدام المطلوبة:

1. Input Picker Mode:
المستخدم يضغط على input، يظهر التقويم، يختار التاريخ، ثم يتم تعبئة input بقيمة العرض حسب التقويم المختار.

مثال:
<input type="text" id="date_display">

بعد اختيار تاريخ هجري:
1447-01-12

يظهر في input:
1447-01-12

2. Hidden Output Mode:
بعد اختيار التاريخ، يتم وضع القيمة الميلادية النهائية في input مخفي ليتم إرسالها لقاعدة البيانات.

مثال:
<input type="text" id="date_display">
<input type="hidden" id="date_value" name="date">

قيمة العرض:
1447-01-12

قيمة الحقل المخفي:
2025-06-08

3. Visible Output Result Mode:
بعد اختيار التاريخ، يتم عرض النتيجة المحولة في عنصر ظاهر للمستخدم.

مثال:
<input type="text" id="date_display">
<div id="date_result"></div>

بعد اختيار تاريخ هجري:
1447-01-12

يظهر داخل div:
التاريخ الميلادي: 2025-06-08

4. Converter Mode:
يجب أن يمكن استخدام المكتبة كأداة تحويل تقاويم في صفحة تجربة.
المستخدم يختار:
- نوع التقويم المستخدم في الإدخال
- التاريخ
- صيغة الإخراج
- اللغة
ثم تظهر له النتيجة المحولة مباشرة.

طريقة الاستخدام المطلوبة عبر HTML:

<link rel="stylesheet" href="https://cdn.example.com/multi-calendar-datepicker.min.css">
<script src="https://cdn.example.com/multi-calendar-datepicker.min.js"></script>

<input
  type="text"
  id="birth_date_display"
  data-mcd
  data-calendar="hijri"
  data-output="#birth_date"
  data-result="#birth_date_result"
  placeholder="اختر التاريخ">

<input type="hidden" id="birth_date" name="birth_date">
<div id="birth_date_result"></div>

<script>
  MultiCalendarDatepicker.init();
</script>

بعد اختيار المستخدم لتاريخ هجري مثل:
1447-01-12

يجب أن تكون قيمة الحقل المخفي:
2025-06-08

ويجب أن يظهر في عنصر النتيجة:
التاريخ الميلادي: 2025-06-08

ملاحظة:
القيمة النهائية المرسلة للسيرفر يجب أن تكون دائمًا ميلادية Gregorian بصيغة YYYY-MM-DD، بغض النظر عن التقويم المستخدم في العرض.

واجهات الاستخدام JavaScript API:

1. التهيئة العامة:

MultiCalendarDatepicker.init();

2. تهيئة عنصر محدد:

MultiCalendarDatepicker.init('#my_date', {
  calendar: 'hijri',
  locale: 'ar',
  dir: 'rtl',
  outputFormat: 'YYYY-MM-DD',
  outputTarget: '#my_hidden_date',
  resultTarget: '#my_visible_result',
  resultTemplate: 'التاريخ الميلادي: {{gregorian}}'
});

3. إنشاء Datepicker برمجيًا:

const picker = new MultiCalendarDatepicker('#date_input', {
  calendar: 'gregorian',
  locale: 'en'
});

4. الحصول على القيمة الميلادية:

picker.getGregorianValue();

يعيد:
"2026-06-27"

5. الحصول على قيمة العرض:

picker.getDisplayValue();

6. الحصول على كائن النتيجة كاملًا:

picker.getResult();

يعيد مثالًا:
{
  calendar: 'hijri',
  displayDate: {
    year: 1447,
    month: 1,
    day: 12
  },
  gregorianDate: {
    year: 2025,
    month: 6,
    day: 8
  },
  gregorianValue: '2025-06-08',
  displayValue: '1447-01-12',
  locale: 'ar'
}

7. تعيين قيمة ميلادية:

picker.setGregorianValue('2026-06-27');

ويجب أن يقوم بتحويلها وعرضها حسب التقويم المحدد.

8. تغيير التقويم:

picker.setCalendar('hijri');

9. تحديث عنصر النتيجة يدويًا:

picker.updateResult();

10. مسح النتيجة:

picker.clear();

11. تدمير العنصر:

picker.destroy();

{
  calendar: 'gregorian' | 'hijri' | 'jalali',
  hijriMode: 'ummalqura' | 'tabular',

  dir: 'rtl' | 'ltr',
  outputFormat: 'YYYY-MM-DD',
  displayFormat: 'YYYY-MM-DD',

}

شرح خصائص الإخراج:

outputTarget:
عنصر hidden input أو input عادي يتم وضع القيمة الميلادية النهائية داخله.
مثال:
outputTarget: '#date_value'

resultTarget:
عنصر ظاهر يتم عرض النتيجة المحولة داخله.
يمكن أن يكون:
- div
- span
- p
- textarea
- input readonly

مثال:
resultTarget: '#date_result'

resultTemplate:
قالب نصي لعرض النتيجة.

أمثلة:
resultTemplate: 'التاريخ الميلادي: {{gregorian}}'
resultTemplate: 'Selected: {{display}} | Gregorian: {{gregorian}}'
resultTemplate: '{{calendar}} => {{gregorian}}'

المتغيرات المدعومة داخل resultTemplate:
{{gregorian}}
{{display}}
{{calendar}}
{{year}}
{{month}}
{{day}}
{{locale}}

مثال:
إذا كان:
display = 1447-01-12
gregorian = 2025-06-08
calendar = hijri

وكان القالب:
'تم اختيار {{display}}، والميلادي المقابل هو {{gregorian}}'

فالنتيجة:
تم اختيار 1447-01-12، والميلادي المقابل هو 2025-06-08

التقاويم المطلوبة في النسخة الأولى:

1. Gregorian Calendar
- التقويم الميلادي.
- هو التقويم الأساسي للإخراج النهائي.
- يجب أن يدعم السنوات والشهور والأيام.
- يجب أن يدعم leap years.

2. Hijri Calendar
- التقويم الهجري.
- يجب دعم وضعين:
  أ. tabular
  ب. ummalqura إن أمكن
- إذا كان دعم ummalqura الكامل يحتاج بيانات، قم بإنشاء هيكل adapter يسمح بإضافة بيانات Umm al-Qura لاحقًا.
- يجب توثيق الفرق بين tabular و ummalqura.
- يجب تحويل التاريخ الهجري إلى ميلادي والعكس.

3. Jalali / Persian Calendar
- التقويم الفارسي.
- يجب تحويله إلى ميلادي والعكس.
- يجب دعم أسماء الأشهر الفارسية لاحقًا، لكن في البداية يكفي أسماء عربية/إنجليزية أو أرقام.

مبدأ مهم:
لا تستخدم JavaScript Date داخليًا للتاريخ إلا عند الحاجة القصوى. تعامل مع التاريخ كـ DateOnly object لتجنب مشاكل timezone.

صيغة DateOnly:

{
  year: 2026,
  month: 6,
  day: 27
}

أي تحويل أو مقارنة يجب أن يعتمد على DateOnly وليس DateTime.

واجهة CalendarAdapter:

كل تقويم يجب أن يطبق هذه الدوال:

class CalendarAdapter {
  id;
  name;

  toGregorian(dateOnly) {}
  fromGregorian(gregorianDateOnly) {}
  getMonthLength(year, month) {}
  getMonths(locale) {}
  getWeekdays(locale) {}
  isLeapYear(year) {}
  validateDate(dateOnly) {}
  getToday() {}
}

مثال:

HijriCalendar.toGregorian({
  year: 1447,
  month: 1,
  day: 12
});

يعيد:
{
  year: 2025,
  month: 6,
  day: 8
}

نظام ResultRenderer:

يجب إنشاء وحدة مستقلة باسم ResultRenderer مسؤولة عن عرض النتيجة في resultTarget.

المهام:
1. استقبال بيانات النتيجة.
2. تطبيق resultTemplate.
3. تحديث محتوى العنصر.
4. دعم العناصر التالية:
   - div
   - span
   - p
   - textarea
   - input
5. إذا كان العنصر input أو textarea يتم تحديث value.
6. إذا كان العنصر div أو span أو p يتم تحديث textContent.
7. عدم استخدام innerHTML لتجنب XSS.
8. دعم مسح النتيجة عند clear.

مثال:

ResultRenderer.render('#date_result', {
  gregorian: '2025-06-08',
  display: '1447-01-12',
  calendar: 'hijri',
  locale: 'ar'
}, 'التاريخ الميلادي: {{gregorian}}');

يجب أن ينتج:
التاريخ الميلادي: 2025-06-08

السلوك المطلوب:
1. عند الضغط على input يظهر التقويم.
2. عند اختيار تاريخ:
   - يتم عرضه في input حسب التقويم المختار.
   - يتم تحويله إلى ميلادي.
   - يتم وضع القيمة الميلادية في outputTarget إذا كان موجودًا.
   - يتم عرض النتيجة في resultTarget إذا كان موجودًا.
   - يتم تحديث Result Preview إذا كان مفعلاً.
   - يتم إطلاق حدث change.
   - يتم إطلاق حدث mcd:change مع التفاصيل.
3. عند الضغط خارج التقويم يتم الإغلاق.
4. عند الضغط على Escape يتم الإغلاق.
5. عند اختيار Today يتم اختيار تاريخ اليوم حسب التقويم النشط.
6. عند Clear يتم مسح input والـ outputTarget والـ resultTarget.
7. يجب دعم أكثر من Datepicker في نفس الصفحة.

    let picker = null;

    const displayInput = document.querySelector('#demo_date_display');
    const hiddenInput = document.querySelector('#demo_date_value');
    const resultBox = document.querySelector('#demo_date_result');
    const eventOutput = document.querySelector('#demo_event_output');

    function getSettings() {
      return {
        calendar: document.querySelector('#demo_calendar').value,
        locale: document.querySelector('#demo_locale').value,
        dir: document.querySelector('#demo_dir').value,
        displayFormat: document.querySelector('#demo_format').value,
        outputFormat: 'YYYY-MM-DD',
        digits: document.querySelector('#demo_digits').value,
        theme: document.querySelector('#demo_theme').value,
        outputTarget: '#demo_date_value',
        resultTarget: '#demo_date_result',
        resultTemplate: 'التاريخ المعروض: {{display}} | التاريخ الميلادي لقاعدة البيانات: {{gregorian}}',
        showResult: true,
        showTodayButton: true,
        showClearButton: true
      };
    }

    function applyPageDirection(settings) {
      document.documentElement.lang = settings.locale;
      document.documentElement.dir = settings.dir;
      document.body.classList.toggle('demo-dark', settings.theme === 'dark');
    }

    function initDemo() {
      const settings = getSettings();

      applyPageDirection(settings);

      if (picker && typeof picker.destroy === 'function') {
        picker.destroy();
      }

      displayInput.value = '';
      hiddenInput.value = '';
      resultBox.textContent = 'لم يتم اختيار تاريخ بعد.';
      eventOutput.textContent = '{}';

      picker = new MultiCalendarDatepicker('#demo_date_display', settings);

      displayInput.addEventListener('mcd:change', function(e) {
        eventOutput.textContent = JSON.stringify(e.detail, null, 2);
      });
    }

    document.querySelector('#demo_apply').addEventListener('click', initDemo);

    document.querySelector('#demo_clear').addEventListener('click', function() {
      if (picker) {
        picker.clear();
      }
      eventOutput.textContent = '{}';
    });

    document.querySelector('#demo_set').addEventListener('click', function() {
      if (picker) {
        picker.setGregorianValue('2026-06-27');
      }
    });

    document.querySelector('#demo_get').addEventListener('click', function() {
      if (picker) {
        const data = picker.getResult();
        eventOutput.textContent = JSON.stringify(data, null, 2);
      }
    });

    initDemo();
  </script>

اختبارات مطلوبة:

1. Gregorian leap year:
- 2024-02-29 صحيح
- 2023-02-29 غير صحيح

2. Format:
- DateOnly 2026-06-27 إلى YYYY-MM-DD يعيد 2026-06-27

3. Parse:
- 2026-06-27 يعيد {year:2026, month:6, day:27}

4. Arabic digits:
- ٢٠٢٦-٠٦-٢٧ تتحول داخليًا إلى 2026-06-27

5. Output hidden:
- عند اختيار تاريخ، hidden input يحتوي قيمة Gregorian.

6. Visible result:
- عند اختيار تاريخ، resultTarget يحتوي النص الناتج حسب resultTemplate.

7. Multiple instances:
- أكثر من input في نفس الصفحة يعمل بدون تعارض.

8. Min/Max:
- لا يسمح باختيار تاريخ خارج الحدود.

9. Events:
- mcd:change يتم إطلاقه بالتفاصيل الصحيحة.

10. Demo page:
- examples/demo.html تعمل عند فتحها في المتصفح.
- يمكن تغيير التقويم واللغة والاتجاه.
- يمكن اختيار تاريخ ورؤية النتيجة الميلادية.
- يمكن قراءة JSON للحدث.

مهم جدًا بخصوص التحويلات:
إذا كانت خوارزميات Hijri أو Jalali تحتاج دقة عالية، استخدم خوارزميات معروفة وموثوقة.
لا تستخدم مكتبات ضخمة إلا إذا كانت ضرورية.
إذا استخدمت مكتبة خارجية، اجعلها اختيارية أو مدمجة بطريقة لا تزيد الحجم كثيرًا.

الأمان:
- لا تستخدم innerHTML مع قيم المستخدم إلا بعد sanitization.
- لا تنفذ أي كود من attributes.
- تجنب XSS.
- استخدم textContent أو value عند عرض النتائج.
